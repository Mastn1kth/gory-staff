КАК ЗАПУСТИТЬ ПРОЕКТ GORY STAFF НА ДРУГОМ КОМПЬЮТЕРЕ
=====================================================

Обновлено: 2026-05-30

Коротко по факту
----------------

Человек может скачать проект с GitHub и запустить его на другом Windows-компьютере,
но это не формат "скачал и вообще ничего не устанавливал".

На другом компьютере обязательно нужны:
1. Node.js LTS.
2. Docker Desktop.
3. Интернет для первой установки зависимостей npm.
4. Git или скачанный ZIP с GitHub.

Из GitHub специально не передаются:
- server\.env с реальными паролями и секретами;
- текущая PostgreSQL база;
- data\Gory-Data.xlsx;
- backups\*.sql;
- runtime-логи и pid-файлы;
- node_modules;
- APK/build-артефакты, если они не переданы отдельно;
- реальные ключи iiko, VK, Instagram, Firebase.

Значит, после скачивания с GitHub получится чистый запуск / демо-база.
Если нужна точная рабочая база ресторана, надо отдельно перенести backup базы и,
при необходимости, server\.env.


ВАРИАНТ 1. ЧИСТЫЙ ЗАПУСК С GITHUB
=================================

Этот вариант подходит, если нужно поднять проект на новом ПК без текущих данных
ресторана. База создается заново.


1. Установить Node.js LTS
-------------------------

1. Открыть сайт:
   https://nodejs.org/
2. Скачать версию LTS для Windows.
3. Установить с обычными настройками.
4. После установки открыть новое окно PowerShell и проверить:

   node -v
   npm -v

Если команды не найдены, закрыть и снова открыть PowerShell.
Если все равно не найдены, Node.js установился без добавления в PATH.


2. Установить Docker Desktop
----------------------------

1. Открыть сайт:
   https://www.docker.com/products/docker-desktop/
2. Скачать Docker Desktop for Windows.
3. Установить.
4. Перезагрузить компьютер, если установщик просит.
5. Открыть Docker Desktop один раз вручную.
6. Дождаться, пока Docker покажет, что он запущен.

Проверка в PowerShell:

   docker --version
   docker info

Если docker info падает, значит Docker Desktop еще не запущен или не готов.


3. Скачать проект с GitHub
--------------------------

Вариант A - через ZIP:

1. Открыть:
   https://github.com/Mastn1kth/gory-staff
2. Нажать Code -> Download ZIP.
3. Распаковать ZIP, например в:
   C:\Users\user\Desktop\Gor Staff

Важно: лучше не класть проект в путь с очень длинным именем или в OneDrive,
чтобы не ловить лишние проблемы с правами и путями.

Вариант B - через Git:

   cd C:\Users\user\Desktop
   git clone https://github.com/Mastn1kth/gory-staff.git "Gor Staff"
   cd "Gor Staff"


4. Установить зависимости
-------------------------

Открыть PowerShell в папке проекта:

   cd "C:\Users\user\Desktop\Gor Staff"
   npm install

Это скачает node_modules. Первый запуск может занять несколько минут.

Если npm install падает:
- проверить интернет;
- проверить, что Node.js установлен;
- удалить полусозданную папку node_modules и повторить npm install.


5. Создать server\.env
----------------------

Самый простой вариант - не создавать его вручную.
Батник START_GORY_STAFF.bat сам создает server\.env с новыми случайными секретами.

Если нужно сделать вручную копию примера:

   Copy-Item server\.env.example server\.env

Но в server\.env.example стоят placeholder-значения. Для рабочего сервера
JWT_SECRET и GUEST_JWT_SECRET должны быть длиннее 32 символов и разными.

Проще запустить START_GORY_STAFF.bat: он сам заполнит сильные случайные секреты.


6. Запустить сервер одним файлом
--------------------------------

Самый простой способ:

1. Открыть папку проекта.
2. Запустить:
   Горы Управление.exe
3. В панели нажать:
   Запустить сервер

Панель/батник сделают:
- проверку Node.js;
- проверку Docker Desktop;
- npm install, если нет node_modules;
- создание/обновление server\.env;
- запуск PostgreSQL в Docker;
- применение таблиц и миграций;
- запуск backend на порту 4000;
- запуск public relay/iiko connector, если они нужны и настройки есть.

Альтернативно можно запустить без панели:

   tools\bat\START_GORY_STAFF.bat


7. Проверить, что сервер работает
---------------------------------

В браузере на том же компьютере открыть:

   http://localhost:4000/health

Должен быть JSON-ответ с ok: true.

В панели "Горы Управление" можно нажать:

   Проверить сервер


8. Логин и пароль
-----------------

При первом запуске START_GORY_STAFF.bat создает server\.env.
Внутри него будут:

   INITIAL_MANAGER_LOGIN
   INITIAL_MANAGER_PASSWORD
   DEMO_STAFF_PASSWORD

Если server\.env не переносился со старого ПК, логин/пароли будут новыми.
Их нужно смотреть на новом ПК в файле:

   server\.env

Не отправлять server\.env случайным людям: там секреты сервера и пароли.


9. Телефон в той же Wi-Fi сети
------------------------------

Сервер работает на компьютере на порту 4000.
Телефон должен быть в той же Wi-Fi сети.

Нужно знать IP компьютера, например:

   http://192.168.0.10:4000

Проверка с телефона:

   http://IP_КОМПЬЮТЕРА:4000/health

Если не открывается:
- проверить, что телефон и ПК в одной Wi-Fi сети;
- проверить Windows Firewall;
- запустить START_GORY_STAFF.bat от администратора, чтобы он мог создать firewall rule;
- проверить, что server не упал.


10. APK на телефоне
-------------------

GitHub не обязан хранить готовый APK. Если нужно поставить приложение на телефон,
есть два варианта.

Вариант A - APK передали отдельно:

1. Скопировать builds\Gory-latest.apk на новый ПК или прямо на телефон.
2. Установить на телефон.
3. Запустить сервер.

Вариант B - собрать APK заново:

1. Нужна настройка Expo/EAS.
2. Из папки проекта запустить:

   tools\bat\BUILD_ANDROID_APK.bat

Это отдельный процесс, он может потребовать вход в Expo/EAS.


ВАРИАНТ 2. ПЕРЕНЕСТИ РЕАЛЬНУЮ РАБОЧУЮ БАЗУ С ДРУГОГО ПК
========================================================

Этот вариант нужен, если на новом компьютере должны быть те же сотрудники, гости,
бонусы, бронирования, столы, истории и другие реальные данные.


1. На старом ПК сделать backup базы
-----------------------------------

Самый простой способ:

1. Открыть Горы Управление.exe на старом ПК.
2. Нажать:
   Бэкап базы
3. Проверить, что в папке backups появился свежий файл:
   gory_*.sql

Если нужно вручную:

   tools\bat\BACKUP_GORY_DATABASE.bat


2. Что скопировать на новый ПК
------------------------------

Обязательно:

- весь проект из GitHub;
- последний backup:
  backups\gory_*.sql

По ситуации:

- server\.env, если нужны те же пароли/секреты/iiko/social settings;
- data\Gory-Data.xlsx, если нужен текущий Excel;
- builds\Gory-latest.apk, если нужен готовый APK.

Не нужно копировать:

- node_modules;
- runtime;
- mobile\android\build;
- mobile\dist;
- логи;
- старые pid-файлы.


3. Восстановить базу на новом ПК
--------------------------------

1. Установить Node.js LTS.
2. Установить Docker Desktop.
3. Скачать/распаковать проект.
4. Скопировать backup в папку:

   backups\

5. Запустить:

   Горы Управление.exe

6. В панели нажать:

   Восстановить

Панель возьмет самый свежий непустой .sql из backups.

Альтернативно:

   tools\bat\RESTORE_GORY_DATABASE.bat

После восстановления запустить server:

   tools\bat\START_GORY_STAFF.bat

И проверить:

   http://localhost:4000/health


4. Если перенесен server\.env
-----------------------------

Если скопировали server\.env со старого ПК, логин/пароли останутся теми же.
Это удобно, но опасно: server\.env нельзя давать посторонним.

Если server\.env не скопирован, новый ПК создаст новые секреты и новые пароли.
Тогда старые данные базы могут остаться, но доступы нужно проверить отдельно.


IIKO, SOCIAL IMPORT, PUSH
=========================

По умолчанию после скачивания с GitHub реальные ключи не настроены.

Для iiko нужно заполнить в server\.env:

   IIKO_ENABLED=true
   IIKO_API_LOGIN=...
   IIKO_ORGANIZATION_ID=...
   IIKO_TERMINAL_GROUP_ID=...
   IIKO_WEBHOOK_SECRET=...

Для social import:

   INSTAGRAM_ACCESS_TOKEN=...
   INSTAGRAM_BUSINESS_ACCOUNT_ID=...
   VK_ACCESS_TOKEN=...
   VK_GROUP_ID=...

Без этих ключей server запустится, но интеграции будут отключены или будут писать,
что credentials не заданы.


PUBLIC DOMAIN И МОБИЛЬНЫЙ ИНТЕРНЕТ
==================================

В APK основной адрес указан как:

   https://app.gory-staff.ru

Но сам по себе новый компьютер не становится доступным из интернета просто от того,
что проект скачали с GitHub.

Чтобы телефон работал через мобильный интернет, нужен рабочий public relay/domain.
В проекте есть Cloudflare relay код, но его реальная работа зависит от настройки
домена, Cloudflare Worker и локального соединения с компьютера ресторана.

Для первой проверки на другом ПК лучше проверить локально:

   http://localhost:4000/health

Потом с телефона в той же Wi-Fi:

   http://IP_КОМПЬЮТЕРА:4000/health

И только потом разбирать public domain.


ЧАСТЫЕ ПРОБЛЕМЫ
===============

1. "Node.js was not found"
--------------------------

Node.js не установлен или PowerShell открыт до установки.
Установить Node.js LTS и открыть новое окно PowerShell.


2. "Docker Desktop was not found"
---------------------------------

Docker Desktop не установлен.
Установить Docker Desktop и запустить его один раз вручную.


3. "Docker Desktop is not ready"
--------------------------------

Docker установлен, но еще не запущен.
Открыть Docker Desktop и подождать 1-5 минут.


4. "PostgreSQL did not become ready in time"
--------------------------------------------

Docker долго стартует или контейнер не поднялся.
Проверить Docker Desktop, потом повторить START_GORY_STAFF.bat.


5. "Port 4000 busy"
-------------------

Уже запущен другой server.
Нужно остановить:

   tools\bat\STOP_GORY_STAFF.bat

Потом снова запустить:

   tools\bat\START_GORY_STAFF.bat


6. Телефон не видит server в Wi-Fi
----------------------------------

Проверить:

- телефон и ПК в одной Wi-Fi сети;
- Windows Firewall разрешает порт 4000;
- server открывается на ПК по http://localhost:4000/health;
- на телефоне открывается http://IP_КОМПЬЮТЕРА:4000/health.

Если firewall мешает, запустить START_GORY_STAFF.bat от администратора.


7. Логин не подходит
--------------------

Открыть server\.env на том ПК, где запущен server.
Проверить:

   INITIAL_MANAGER_LOGIN
   INITIAL_MANAGER_PASSWORD
   DEMO_STAFF_PASSWORD

Если база была восстановлена из backup, а server\.env другой, доступы могут не
совпадать с ожиданиями. Для переноса "один в один" нужно переносить и backup,
и server\.env только доверенному человеку.


МИНИМАЛЬНАЯ КОМАНДА ДЛЯ РАЗРАБОТЧИКА
=====================================

Если человек не пользуется панелью и делает все через terminal:

   cd "C:\Users\user\Desktop\Gor Staff"
   npm install
   Copy-Item server\.env.example server\.env
   docker compose up -d
   npm --workspace server run db:init
   npm run server

Но для обычного пользователя лучше:

   Горы Управление.exe

или:

   tools\bat\START_GORY_STAFF.bat


ГЛАВНЫЙ ВЫВОД
=============

С GitHub можно скачать исходники и запустить проект на другом ПК.
Но для рабочей копии ресторана нужно отдельно перенести:

1. Backup базы из backups.
2. server\.env, если нужны те же пароли и интеграции.
3. APK, если нужна установка на телефон без новой сборки.

Без этого будет чистая установка, а не копия текущего ресторана.
