# Перенос проекта на другой ПК

Коротко: можно отправить папку проекта целиком, но это не настоящий standalone-установщик. На другом Windows-ПК всё равно нужны Docker Desktop и Node.js. `Горы Управление.exe` сам по себе сервер не содержит, он только открывает панель `gory-control\GoryControl.ps1` из этой же папки.

## Что обязательно должно быть в папке

- `Горы Управление.exe`
- `gory-control\`
- `tools\`
- `server\`
- `mobile\`
- `cloudflare\`
- `docker-compose.yml`
- `package.json`
- `package-lock.json`
- `builds\Gory-latest.apk`, если телефон должен ставить уже собранное приложение
- `data\Gory-Data.xlsx`, если нужен текущий Excel
- `backups\*.sql`, если нужно перенести текущую PostgreSQL-базу
- `server\.env`, если доверенному человеку нужны те же серверные секреты и логины

`node_modules` отправлять не обязательно. Если его нет, кнопка запуска сама выполнит `npm install`. Для этого на другом ПК нужен интернет.

## Что не переносится простым копированием папки

Текущая PostgreSQL-база живёт в Docker volume, а не в папке проекта. Поэтому копия папки без SQL-бэкапа поднимет новый контейнер и получит новую/демо-базу.

Перед передачей нормальный порядок такой:

1. На исходном ПК открыть `Горы Управление.exe`.
2. Нажать `Бэкап базы`.
3. Убедиться, что в `backups` появился свежий непустой файл `gory_*.sql`.
4. Отправить папку проекта вместе с этим файлом.

## Первый запуск на другом ПК

1. Установить Node.js LTS.
2. Установить Docker Desktop.
3. Открыть Docker Desktop один раз и дождаться, что он запущен.
4. Распаковать проект, лучше без переименования главной папки.
5. Запустить `Горы Управление.exe` из корня проекта.
6. Если нужна твоя текущая база, нажать в панели `Восстановить`. Панель возьмёт самый свежий непустой `.sql` из `backups`. То же самое можно сделать вручную через `tools\bat\RESTORE_GORY_DATABASE.bat`.
7. В `Горы Управление.exe` нажать `Запустить сервер`.
8. Нажать `Проверить сервер`.
9. Для телефона установить `builds\Gory-latest.apk`.

## Логины и пароли

Если `server\.env` передан, логин управляющего и пароли будут теми же, что записаны в этом файле. Не отправляй `server\.env` случайным людям.

Если `server\.env` не передан, первый запуск создаст новый файл `server\.env` со случайными паролями. Тогда их надо смотреть уже на новом ПК в `server\.env`.

## Честное ограничение

Формат "человек скачал один файл и вообще ничего не устанавливал" сейчас не сделан. Для этого надо отдельно собирать установщик или переносить сервер на VPS/облако, чтобы на чужом ПК не требовались Docker и Node.js.
