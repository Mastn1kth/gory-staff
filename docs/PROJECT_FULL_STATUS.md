# Горы: полный состав проекта

Этот файл нужен как понятное описание того, что уже есть в проекте приложения «Горы».

## Главная схема

- Компьютер владельца работает как сервер ресторана.
- На компьютере запускаются backend, PostgreSQL и локальный HTTPS relay к Cloudflare.
- Телефон гостя или сотрудника устанавливает Android APK.
- APK подключается к серверу через публичный HTTPS-адрес или локальный Wi-Fi.
- Основной адрес: `https://app.gory-staff.ru`.
- Данные хранятся на сервере в PostgreSQL.
- Телефон хранит кэш последних данных и очередь офлайн-действий.

## Батники

- `START_GORY_STAFF.bat` запускает Docker, PostgreSQL, backend, публичный HTTPS relay и проверяет адреса.
- `STOP_GORY_STAFF.bat` останавливает сервер, PostgreSQL-контейнер, публичный relay, Metro/Expo и Gradle daemons.
- `START_PUBLIC_RELAY.bat` запускает исходящую связь компьютера ресторана с Cloudflare для домена `https://app.gory-staff.ru`.
- `BUILD_ANDROID_APK.bat` собирает свежий Android APK локально через Android Studio / Gradle.

## Android APK

- Готовый файл: `Gory-latest.apk`.
- Название приложения на телефоне: «Горы».
- Основной сервер внутри APK: `https://app.gory-staff.ru`.
- Для Wi-Fi теста можно собрать APK с локальным адресом через `BUILD_ANDROID_APK.bat wifi`.
- `localhost` и `127.0.0.1` не используются для реального телефона.

- Гостевое меню без корзины, доставки, заказов и оплаты.
- Отдельный раздел «Как добраться».
- Кнопки маршрута, звонка и копирования адреса.

- Регистрация гостя по имени и телефону.
- Вход гостя по телефону.
- Бонусная карта.
- Баланс бонусов.
- История бонусов.
- Реферальный код.
- Реферальная регистрация.
- Кэш гостевого профиля, меню и бонусов.

## Клиентская база

- Гости отделены от сотрудников.
- Есть таблицы гостей, карт, бонусных операций, рефералов, сессий, устройств, согласий, заметок и сегментов.
- У гостя есть телефон, бонусы, уровень лояльности и реферальный код.
- Бонусный баланс считается через операции, а не простым ручным числом.
- Предусмотрены уровни: бронза, серебро, золото, платина.
- При регистрации начисляется стартовый бонус.
- Реферальный бонус защищён от повторного начисления.

## Рабочая зона персонала

- Рабочая зона скрыта внутри гостевого профиля через кнопку «Для сотрудников».
- Доступ управляющего задаётся через `INITIAL_MANAGER_LOGIN` и `INITIAL_MANAGER_PASSWORD` в `server\.env`; демо-пароли не показываются в приложении.
- Роли и права доступа сохранены.
- Управляющий видит админские разделы.
- Официант видит рабочие разделы официанта.
- Хостес видит план зала и брони.
- Кухня и бар видят свои рабочие разделы.
- Чат из базы не удалён, но в интерфейсе не показывается.

## Онлайн через мобильный интернет

- Поддержан публичный адрес `https://app.gory-staff.ru` через Cloudflare Worker и локальный HTTPS relay.
- APK должен работать через мобильный интернет, если ПК-сервер и локальный relay запущены.
- Обычный Cloudflare Tunnel на текущем интернет-канале не используется, потому что его постоянное соединение обрывалось при проверках.
- В батнике показываются адреса для ПК, Wi-Fi и мобильного интернета.
- В приложении есть диагностика API URL, WebSocket, сервера и push.

## Push-уведомления

- Добавлена база под push-устройства.
- Есть таблицы уведомлений, настроек и delivery log.
- Телефон может получить push token.
- Token привязывается к гостю или сотруднику.
- Есть тестовое push-уведомление.
- Секретные ключи Firebase не хардкодятся.

## Offline-first

- Приложение открывается без интернета, если данные уже загружались.
- Сессия гостя не сбрасывается при потере сети.
- Сессия сотрудника не сбрасывается при потере сети.
- Меню, профиль, бонусы, рабочие данные и план зала показываются из кэша.
- Без интернета не показывается пустой экран.
- Есть статус «Нет соединения».
- Есть автоматическое восстановление подключения.
- Есть очередь офлайн-действий.
- Разрешённые офлайн-действия сохраняются и отправляются позже.
- Критичные действия без сервера блокируются.

## Что нельзя делать без интернета

- Новый вход, если пользователь раньше не входил на телефоне.

- Изменение ролей.
- Блокировка сотрудников.
- Удаление данных.
- Критичные админские действия.
- Push-отправка.

## Что можно делать без интернета

## Последний большой UX-проход

- Гостевая часть уже работает в 4 вкладках: Профиль, Главная, Меню, Маршрут.
- Реферальный код открывается как отдельная карточка с QR-заглушкой, копированием и отправкой.
- Уровень лояльности открывает отдельное окно с уровнями и преимуществами.
- Тестовый push больше не является обычным блоком гостевого профиля; проверка push остаётся в диагностике.
- Вход сотрудников получил быстрые dev-кнопки и форму, которая поднимается над клавиатурой.
- Главный рабочий экран управляющего стал чище: пульс смены без лишней карточки имени.
- Раздел клиентов очищен от общего блока последних операций; история бонусов показывается при работе с выбранным клиентом.
- У сотрудника есть редактирование профиля и выбор аватарки-заглушки.
- У официанта появился блок `Акценты продаж` в меню.
- У хостес есть рабочие блоки `Сегодня`, `Зал`, `Брони` и `Ожидание`.
- Сервер создаёт уведомления по бонусам, рефералам, броням, листу ожидания, стоп-листу и задачам.
- Корневая папка визуально очищена: на виду только батники, README и актуальный APK.
